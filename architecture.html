BYOM — Bring Your Own Model

Why BYOM changes
everything for enterprise AI.

Cloud AI SaaS forces you into their model. Drayvn doesn't.

01
Connect Any Model Provider
Add your OpenAI API key, Anthropic key, Google AI key, or any OpenAI-compatible endpoint. Drayvn routes calls through its governed layer — PII stripped, costs tracked, policies enforced.
02
Host Your Own Private Model
Load Llama-3, Mistral, Phi-3, or your fine-tuned model onto the Dell Pro Max in your office. It serves inference requests at zero marginal cost, 24/7, to any agent that needs it.
03
Route by Policy, Not by Hand
Define rules: HR queries → on-prem model. Customer-facing tasks → GPT-4o. Legal document analysis → Claude 3.5. Model routing is a policy, not a config file you maintain.
04
Switch Without Re-Engineering
Model-agnostic orchestration means you can swap any model without touching agent code. When GPT-5 drops, you flip a routing rule. Your agents keep running.
BYOM Model Routing Policy
If data is PHI or PII
→ Dell Pro Max (Llama-3) — never leaves building
If task is domain-specific
→ Your custom fine-tuned model (Model Studio)
If task is general reasoning
→ GPT-4o / Claude 3.5 (PII stripped first)
If cost budget exceeded
→ Route to cheaper model or block + alert
All routing is audited, logged, and reversible
Data Flow — Sensitive Query

What happens when an employee
asks a sensitive question.

Follow a query containing PII through the Drayvn stack. Every step is governed, logged, and auditable.

1
Employee submits query via Enterprise Copilot
"Show me John Smith's (SSN: 234-56-7890) claim history from the past 6 months."
Origin: HR Department · User: Sarah Chen
2
Edge-Side Sentinel intercepts before network
On the Dell Pro Max in your building, The Sentinel detects PII: SSN pattern found. Redacts it to "SSN: [REDACTED-XXX-XX-7890]" and tags the query for on-prem routing.
Data never left your building yet
3
BYOM Routing Policy evaluates
Policy engine detects "PHI/PII" tag → routes to on-premise Llama-3 model on Dell Pro Max. Public cloud APIs are not called for this query.
Policy: PHI → On-Prem model
4
RAG retrieval from private knowledge base
The orchestrator queries the Vector Store (also on-prem) for relevant claims data. Retrieved documents stay within the local network boundary.
Retrieval: On-prem vector DB
5
Local LLM generates response
Llama-3 on Dell Pro Max synthesizes the answer from retrieved claims data. Full response generated on your hardware. Zero external API calls made.
Inference: $0.00 — local hardware
6
Sentinel output check + Observability log
Response passes output scan. Full trace logged to Langfuse: query, model used, tokens, latency, department attribution ($0.00 edge cost), policy decisions.
Logged · Attributed · Auditable · Complete
